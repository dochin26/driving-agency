# コードレビュー

指定されたファイルのコードレビューを行ってください。

## レビュー観点

### 1. Rails規約
- MVC分離は適切か
- 命名規則は正しいか
- RESTful設計に従っているか

### 2. セキュリティ
- SQLインジェクション対策
- XSS対策
- CSRF対策
- Mass Assignment対策（Strong Parameters）
- 認証・認可チェック

### 3. パフォーマンス
- N+1クエリの有無
- 不要なデータベースアクセス
- インデックスの使用
- eager loadingの活用

### 4. 保守性
- コードの可読性
- DRY原則
- コメントの適切さ
- マジックナンバーの排除

### 5. テスタビリティ
- 単体テストが書きやすいか
- 依存関係の注入
- モックしやすい設計

### 6. エラーハンドリング
- 例外処理は適切か
- ユーザーへのエラーメッセージ
- ログ出力

### 7. Rails 8 / 最新機能の活用
- Solid Queue, Solid Cache の活用可能性
- Turbo, Stimulus の適切な使用

## 出力形式

```markdown
## 良い点
- ✅ 〇〇が適切に実装されている

## 改善提案

### 優先度：高
- ⚠️ 問題点の説明
  - 改善案の提示
  - サンプルコード

### 優先度：中
- 💡 改善可能な点

### 優先度：低
- 📝 細かい改善提案
```

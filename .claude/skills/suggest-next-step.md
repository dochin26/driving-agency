# 次のステップ提案

現在の開発状況を確認し、次に取り組むべきタスクを提案してください。

## 確認項目

1. **完了した作業**
   - どこまで実装が進んでいるか
   - マイグレーション、モデル、コントローラー、ビューの状態

2. **未完了の作業**
   - CLAUDE.mdの要件で未実装の機能
   - TODOコメント
   - テストの有無

3. **優先順位**
   - MVP（最小機能）に必要な機能
   - フェーズ2（後回し可能）の機能

4. **依存関係**
   - 先に実装すべき機能
   - 並行して進められる作業

## 提案内容

1. **次の実装タスク**
   - 具体的な作業内容
   - 所要時間の目安

2. **推奨順序**
   - なぜこの順序が良いのか
   - 学習効率の観点

3. **注意点**
   - つまずきやすいポイント
   - 参考資料

## 出力形式

```markdown
## 現在の状況
✅ 完了: 〇〇
🚧 進行中: △△
📋 未着手: ××

## 次のステップ（推奨順）

### 1. [タスク名]（所要時間: X時間）
**内容:**
- 具体的な作業

**理由:**
- なぜこのタスクが重要か

**参考:**
- 関連ドキュメント
- 参考コード

### 2. [次のタスク]
...

## 並行して進められる作業
- タスクA（学習目的）
- タスクB（調査・準備）

## 学習のヒント
- 〇〇を理解するために△△を読む
```
